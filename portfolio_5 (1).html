CHAPTER 01 · UNIT I

Introduction to Statistics & Data

Foundations of data science — understanding data types, measurement scales, and how to collect and represent information meaningfully.

01 — Foundation

📂 Data & Types of Variables

Data refers to raw facts and figures. Every piece of data has a type — understanding the type determines every analysis method you can validly apply.

🟣 Qualitative (Categorical)

Represent labels or categories; cannot be meaningfully added. E.g., gender, blood type, city name.

🟢 Quantitative (Numerical)

Represent counts or measurements; arithmetic is valid. E.g., height, temperature, income.

🔵 Discrete

Countable, whole-number values. E.g., number of students (0, 1, 2, …), dice roll.

🟠 Continuous

Any value within an interval. E.g., height = 170.243… cm, time = 0.378… s.

Population vs Sample: Population (N) — all subjects of interest Sample (n) — subset drawn from population Parameter — numerical summary of population (μ, σ) Statistic — numerical summary of sample (x̄, s)
NominalOrdinalIntervalRatio

📊 Data Type Distribution

Example 1

Classify the variables

📘 Definition

A variable is any characteristic that can take different values across units. Variables are qualitative (categories) or quantitative (numbers, discrete or continuous), measured on Nominal, Ordinal, Interval or Ratio scales.

❓ Example Problem

Classify each: (a) Blood group of a patient (b) Number of cars in a parking lot (c) Temperature in °C (d) Customer-satisfaction rating: Poor / Fair / Good / Excellent.

✅ Solution
(a) Qualitative — Nominal (b) Quantitative — Discrete (Ratio) (c) Quantitative — Continuous (Interval, no true zero) (d) Qualitative — Ordinal
💡 Explanation

Blood group has labels with no order → Nominal. Cars are counted in whole numbers → Discrete. Temperature in °C is continuous but 0°C is not absence of heat → Interval. Ratings have natural order but unequal gaps → Ordinal.

🔍 Interpretation

Identifying the correct scale tells us which statistics are valid: mean is meaningful for (b),(c) but only mode/median for (a),(d). Choosing wrong scale leads to wrong analysis.

02 — Central Tendency

📐 Mean, Median & Mode

Central tendency captures the "centre" of a dataset — a single representative value. Each measure has different mathematical properties and responds differently to outliers.

Mean (x̄)

Sum all values and divide by count — the "balancing point". Sensitive to outliers.

x̄ = Σxᵢ / n Grouped: x̄ = Σ(fᵢ·mᵢ) / Σfᵢ E.g. {2,4,6,8,10} → 30/5 = 6

Median (M)

Middle value when sorted. Robust to outliers — best for skewed data.

Odd n: M = x[(n+1)/2] Even n: M = (x[n/2]+x[n/2+1])/2 Grouped: M = L + [(n/2−F)/f]×h

Mode (Mo)

Most frequent value. Unimodal/bimodal/multimodal. Only valid measure for nominal data.

Czuber: Mo = L + [Δ₁/(Δ₁+Δ₂)]×h Right-skew: Mode < Median < Mean

Weighted Mean

Assigns importance to values — used for GPAs and indices.

x̄_w = Σ(wᵢ·xᵢ) / Σwᵢ GPA: (9×4+7×2+8×3)/(4+2+3)=8.22
Example 2

Mean, Median, Mode of marks

📘 Definition

Central tendency represents the centre of a dataset. Mean = arithmetic average, Median = middle value when sorted, Mode = most frequent value.

❓ Example Problem

Marks of 9 students: 45, 50, 55, 60, 60, 65, 70, 75, 100. Find Mean, Median and Mode.

✅ Solution
n = 9, Σx = 580 Mean = 580 / 9 = 64.44 Median = middle (5th) value = 60 Mode = 60 (occurs twice)
💡 Explanation

Mean is pulled up by the outlier 100. Median is the value at position (n+1)/2 = 5. Mode is the most repeated value.

🔍 Interpretation

Because mean (64.44) > median (60) the data is mildly right-skewed. For skewed data, median is a better summary than mean.

03 — Dispersion

📏 Variance, Std Dev & CV

Two datasets can share identical means yet look completely different. Dispersion measures quantify how spread out the data is around its centre.

Range = x_max − x_min Variance: Population: σ² = Σ(xᵢ − μ)² / N Sample: s² = Σ(xᵢ − x̄)² / (n − 1) (Bessel's correction) Standard Deviation: σ = √σ² s = √s² Empirical Rule (Normal): μ ± 1σ ≈ 68.27% μ ± 2σ ≈ 95.45% μ ± 3σ ≈ 99.73% Coefficient of Variation (dimensionless): CV = (s / x̄) × 100 % Quartiles & IQR: Q1 = 25th, Q2 = 50th (median), Q3 = 75th IQR = Q3 − Q1 Outlier fences (Tukey): [Q1 − 1.5·IQR , Q3 + 1.5·IQR]
⚠ CV not valid when mean ≈ 0 or negative. Use it to compare volatility across different units/scales.
Example 3

Variance, Std Dev and CV

📘 Definition

Dispersion measures how spread the data is. Variance = average squared deviation from mean; SD = √variance; CV = (SD/Mean)×100% — unit-free.

❓ Example Problem

Daily sales (in ₹1000s) for 5 days: 10, 12, 14, 16, 18. Find sample variance, SD and CV.

✅ Solution
x̄ = (10+12+14+16+18)/5 = 14 Deviations: -4,-2,0,2,4 → squared: 16,4,0,4,16 Σ=40 s² = 40/(n-1) = 40/4 = 10 s = √10 ≈ 3.162 CV = (3.162/14)×100 ≈ 22.59 %
💡 Explanation

We use n−1 (Bessel's correction) for the sample variance to get an unbiased estimate. SD is in the same unit as data; CV is dimensionless.

🔍 Interpretation

A CV of ~22.6 % means the daily sales fluctuate by about 23 % around the mean — moderate variability. CV lets us compare volatility of two variables with different units.

04 — Shape

🌀 Skewness

Skewness measures asymmetry of a distribution. Symmetric → 0. Sign and magnitude reveal direction and degree of asymmetry.

Pearson: SK = 3(Mean − Median) / σ Karl Pearson 1st: SK₁ = (Mean − Mode) / σ Moment-based (γ₁): γ₁ = μ₃ / σ³ μ₃ = Σ(xᵢ − x̄)³ / n Interpretation: γ₁ > 0 → Right-skewed (positive) Mean > Median > Mode γ₁ < 0 → Left-skewed (negative) Mean < Median < Mode γ₁ = 0 → Symmetric
  • Right-skewed: long right tail. E.g., income, house prices.
  • Left-skewed: long left tail. E.g., age at death, easy exam scores.

🌀 Skewness Shapes

Example 4

Compute Skewness

📘 Definition

Skewness measures asymmetry. γ₁ > 0 right-skewed (tail on right), γ₁ < 0 left-skewed, γ₁ = 0 symmetric.

❓ Example Problem

For a dataset Mean = 50, Median = 45, SD = 8. Find Pearson's coefficient of skewness and comment.

✅ Solution
SK = 3·(Mean − Median)/σ = 3·(50 − 45)/8 = 15/8 = 1.875
💡 Explanation

Pearson's formula uses the gap between mean and median, scaled by the SD. A large positive value indicates the mean is being pulled to the right.

🔍 Interpretation

SK = +1.875 ⇒ strongly right-skewed distribution (long right tail). Typical examples: income, house prices. Median is a more reliable centre here than the mean.

04b — Quartiles

📐 Quartiles & IQR

Quartiles split sorted data into four equal parts. The interquartile range (IQR = Q3 − Q1) captures the middle 50% and is highly robust to outliers.

📊 Quartile Splits

Example 5

Quartiles and IQR

📘 Definition

Quartiles Q1, Q2, Q3 split sorted data into 4 equal parts. IQR = Q3 − Q1 captures the middle 50% and is robust to outliers.

❓ Example Problem

Data (sorted): 4, 7, 8, 10, 12, 14, 15, 18, 20, 22, 25. Find Q1, Q2, Q3, IQR and outlier fences.

✅ Solution
n = 11 Q2 (median) = 6th value = 14 Lower half: 4,7,8,10,12 → Q1 = 8 Upper half: 15,18,20,22,25 → Q3 = 20 IQR = 20 − 8 = 12 Fences: [Q1 − 1.5·IQR , Q3 + 1.5·IQR] = [-10, 38]
💡 Explanation

Q1 is the median of the lower half, Q3 of the upper half. Tukey's 1.5·IQR fences flag points outside the typical range as potential outliers.

🔍 Interpretation

No data point lies outside [-10, 38] so there are no outliers. The middle 50 % of values lie within a span of 12 units — gives a quick spread summary.

05 — Visualisation

📊 Histogram, Box Plot & QQ Plot

Visual representations transform abstract numbers into immediate insight. Each plot reveals a different structural property of the data.

Histogram

Groups continuous data into equal-width bins; bar heights represent frequency. Sturges' Rule: k ≈ 1 + 3.322·log₁₀(n).

Box Plot

Five-number summary: Min | Q1 | Median | Q3 | Max. Points beyond fences = outliers.

Normal Q–Q Plot

Observed vs theoretical quantiles. Straight diagonal → normally distributed; S-curve → heavy tails.

CHAPTER 02 · UNIT I

Probability & Bayes' Rule

Probability is the language of uncertainty — it quantifies how likely an event is. From classical definitions to Bayes' theorem and Bayesian classification, this chapter builds the foundation for inference and machine learning.

01 — Definition

🎲 What is Probability?

Probability P(A) measures the likelihood of an event A on a 0–1 scale: 0 = impossible, 1 = certain. There are three classical interpretations.

Classical

Equally likely outcomes. P(A) = favourable / total. E.g., P(even on die) = 3/6.

Frequentist

Long-run relative frequency. P(A) = lim (n→∞) f(A)/n. E.g., coin flipped 10⁶ times.

Subjective (Bayesian)

Degree of personal belief, updated as evidence arrives. Foundation of Bayesian inference.

Sample space S — set of all possible outcomes Event A ⊆ S — a subset of interest Axioms: 0 ≤ P(A) ≤ 1 P(S)=1 P(A∪B)=P(A)+P(B) if disjoint
Example 7

Probability of an event

📘 Definition

Probability P(A) is a number in [0,1] expressing how likely event A is. Classical: favourable / total equally-likely outcomes.

❓ Example Problem

A fair die is rolled once. Find P(prime number).

✅ Solution
S = {1,2,3,4,5,6}, |S| = 6 Primes in S = {2,3,5}, |A| = 3 P(prime) = 3/6 = 0.5
💡 Explanation

Each face is equally likely (classical definition). We count favourable outcomes (primes ≤ 6) and divide by total outcomes.

🔍 Interpretation

There is a 50 % chance of getting a prime number on a single roll. Long-run, in 1000 rolls we'd expect ≈ 500 primes (frequentist view).

02 — Rules

➕ Addition & Multiplication Rules

Addition Rule: P(A ∪ B) = P(A) + P(B) − P(A ∩ B) Mutually exclusive: P(A∩B)=0 → P(A∪B) = P(A)+P(B) Multiplication Rule: P(A ∩ B) = P(A) · P(B | A) Independent events: P(A∩B) = P(A)·P(B) Conditional Probability: P(A | B) = P(A ∩ B) / P(B), P(B) > 0 Complement: P(Aᶜ) = 1 − P(A) Total Probability: P(B) = Σᵢ P(B | Aᵢ) · P(Aᵢ)
  • Independent: outcome of one event does not affect the other (e.g., two coin flips).
  • Mutually exclusive: events cannot occur together (e.g., rolling 1 and 6 on one die).
Example 8

Addition & Multiplication rules

📘 Definition

Addition rule: P(A∪B)=P(A)+P(B)−P(A∩B). Multiplication rule: P(A∩B)=P(A)·P(B|A); independent ⇒ P(A∩B)=P(A)·P(B).

❓ Example Problem

A card is drawn from a standard 52-card deck. (a) P(King or Heart)? (b) Two cards are drawn with replacement: P(both Kings)?

✅ Solution
(a) P(K) = 4/52, P(H) = 13/52, P(K∩H) = 1/52 P(K∪H) = 4/52 + 13/52 − 1/52 = 16/52 = 4/13 ≈ 0.308 (b) With replacement → independent P(both K) = (4/52)·(4/52) = 1/169 ≈ 0.00592
💡 Explanation

In (a) the King of Hearts is counted in both sets, so we subtract its probability once. In (b) drawing with replacement makes the two draws independent.

🔍 Interpretation

About 30.8 % chance the card is a King or a Heart. Probability of two consecutive Kings is very small (~0.6 %), showing how independent rare events compound.

03 — Bayes' Theorem

🧠 Bayes' Rule

Bayes' theorem reverses conditional probabilities — given evidence, what is the probability of a hypothesis? It's the engine of modern AI, medical diagnosis, and spam filters.

P(B | A) · P(A) P(A | B) = ───────────────── P(B) Where P(B) = Σᵢ P(B | Aᵢ) · P(Aᵢ) (Law of Total Probability) Terminology: P(A) — Prior (belief before evidence) P(B|A) — Likelihood (evidence given hypothesis) P(A|B) — Posterior (updated belief) P(B) — Evidence / marginal likelihood
💡 Classic example: a 99% accurate test for a disease that affects 1 in 1000 still gives P(disease | positive) ≈ 9% — base rates matter!
Example 9

Disease test (Bayes)

📘 Definition

Bayes' theorem updates a prior belief P(A) into a posterior P(A|B) given evidence B: P(A|B) = P(B|A)·P(A)/P(B).

❓ Example Problem

A disease affects 1 in 1000 people. A test is 99% accurate (sensitivity = specificity = 0.99). A randomly chosen person tests positive. Find P(Disease | Positive).

✅ Solution
P(D)=0.001, P(¬D)=0.999 P(+|D)=0.99, P(+|¬D)=0.01 P(+) = 0.99·0.001 + 0.01·0.999 = 0.00099 + 0.00999 = 0.01098 P(D | +) = (0.99·0.001) / 0.01098 ≈ 0.0902 ≈ 9.02 %
💡 Explanation

Even a highly accurate test produces many false positives when the disease is rare, because the 1 % false-positive rate acts on the huge healthy population (99.9 %).

🔍 Interpretation

Despite a positive result, the actual chance of having the disease is only ~9 %. This is the classic base-rate fallacy — prior probabilities matter enormously.

04 — Application

🤖 Bayesian Classification

The Naive Bayes classifier applies Bayes' theorem with a "naive" independence assumption between features. Despite simplicity, it performs surprisingly well in text classification, spam filtering, and sentiment analysis.

Predict class C* that maximises posterior: C* = argmax_C P(C) · Π P(xᵢ | C) Naive assumption: features x₁,…,xₙ conditionally independent given C. Steps: 1. Compute prior P(C) for each class from training data. 2. Compute likelihood P(xᵢ | C) for each feature. 3. Multiply & pick the class with highest score. Use cases: spam detection, document categorisation, medical diagnosis.

🔔 Bonus: Distribution Preview

Example 10

Naive Bayes spam filter

📘 Definition

Naive Bayes classifies by picking the class C maximising P(C)·∏P(xᵢ|C), assuming features are conditionally independent given the class.

❓ Example Problem

Training: P(Spam)=0.4, P(Ham)=0.6. Word "free": P(free|Spam)=0.8, P(free|Ham)=0.1. Word "meeting": P(meeting|Spam)=0.05, P(meeting|Ham)=0.4. Classify an email containing both words.

✅ Solution
Score(Spam) = 0.4 · 0.8 · 0.05 = 0.0160 Score(Ham) = 0.6 · 0.1 · 0.4 = 0.0240 Normalise: P(Spam|words) = 0.016/(0.016+0.024) = 0.40 P(Ham|words) = 0.024/0.040 = 0.60 → Predict HAM
💡 Explanation

We multiply the prior by each likelihood and pick the larger score. Naive independence lets us multiply word probabilities individually.

🔍 Interpretation

Although "free" is spammy, the presence of "meeting" pushes the email toward Ham. Posterior 60 % Ham vs 40 % Spam → classifier labels it as Ham.

CHAPTER 03 · UNIT II

Random Variables & Probability Distributions

A random variable assigns numerical values to outcomes of random experiments. Distributions describe how probabilities are spread across these values — the building blocks of statistical modelling.

01 — Random Variables

🎯 Discrete vs Continuous

Discrete RV

Takes countable values. Described by Probability Mass Function (PMF). E.g., dice roll, number of emails per hour.

P(X = x) = pₓ, Σ pₓ = 1 E[X] = Σ x · P(X=x) Var(X) = E[X²] − (E[X])²

Continuous RV

Takes any value in an interval. Described by Probability Density Function (PDF). E.g., height, time, voltage.

P(a ≤ X ≤ b) = ∫ₐᵇ f(x) dx ∫ f(x) dx = 1 E[X] = ∫ x · f(x) dx
Example 11

Expectation of a discrete RV

📘 Definition

A discrete RV takes countable values with PMF P(X=x). Expectation E[X] = Σ x·P(X=x); Variance Var(X)=E[X²]−(E[X])².

❓ Example Problem

X = score on a fair die. Find E[X] and Var(X).

✅ Solution
P(X=k)=1/6 for k=1..6 E[X] = (1+2+3+4+5+6)/6 = 21/6 = 3.5 E[X²] = (1+4+9+16+25+36)/6 = 91/6 ≈ 15.167 Var(X) = 15.167 − 3.5² = 15.167 − 12.25 = 2.917 SD = √2.917 ≈ 1.708
💡 Explanation

Each face is equally likely. We sum k·(1/6) for the mean, and k²·(1/6) for E[X²].

🔍 Interpretation

On average a die shows 3.5; typical deviation from this average is about 1.7. These match the well-known formulas (n²−1)/12 for a discrete uniform RV.

02 — Binomial

🎲 Binomial Distribution

Models the number of successes in n independent Bernoulli trials, each with success probability p.

P(X = k) = C(n,k) · pᵏ · (1−p)ⁿ⁻ᵏ, k = 0,1,…,n Mean: μ = np Variance: σ² = np(1−p) Conditions: • Fixed number of trials n • Each trial: success/failure • Constant probability p • Trials independent
Example 12

Binomial: defective items

📘 Definition

Binomial(n,p): number of successes in n independent Bernoulli trials. P(X=k)=C(n,k)·pᵏ·(1−p)ⁿ⁻ᵏ. μ=np, σ²=np(1−p).

❓ Example Problem

A factory produces items with defect probability p = 0.1. A sample of n = 10 items is inspected. Find P(exactly 2 defectives) and the expected number of defectives.

✅ Solution
X ~ Bin(10, 0.1) P(X=2) = C(10,2)·(0.1)²·(0.9)⁸ = 45 · 0.01 · 0.43047 ≈ 0.1937 E[X] = np = 10·0.1 = 1 Var(X) = np(1−p) = 0.9, SD ≈ 0.949
💡 Explanation

C(10,2)=45 ways to pick which 2 of the 10 items are defective; each such arrangement has probability (0.1)²·(0.9)⁸.

🔍 Interpretation

There is ~19.4 % chance of seeing exactly 2 defectives. On average we expect 1 defective per batch of 10 — useful for quality-control planning.

03 — Poisson

📡 Poisson Distribution

Models the number of events occurring in a fixed interval of time/space when events occur independently at a known average rate λ.

P(X = k) = (λᵏ · e⁻ᵏ) / k!, k = 0,1,2,… Mean = Variance = λ Examples: • Calls arriving at a call centre • Defects per page in a book • Radioactive decay events per second Approximation: Binomial(n, p) → Poisson(λ = np) when n is large and p is small.
Example 13

Poisson: call centre arrivals

📘 Definition

Poisson(λ): count of events in fixed interval when events occur independently at average rate λ. P(X=k)=e^{−λ}·λᵏ/k!. Mean = Variance = λ.

❓ Example Problem

A call centre receives an average of λ = 4 calls per minute. Find P(exactly 2 calls in a minute) and P(at least 1 call).

✅ Solution
P(X=2) = e⁻⁴ · 4² / 2! = 0.01832 · 16 / 2 ≈ 0.1465 P(X≥1) = 1 − P(X=0) = 1 − e⁻⁴ = 1 − 0.01832 ≈ 0.9817
💡 Explanation

We plug k=2 into the Poisson PMF. P(at least 1) is found via the complement to avoid an infinite sum.

🔍 Interpretation

~14.7 % chance of exactly 2 calls; ~98.2 % chance of at least one call per minute. Poisson is the go-to model for rare-event counts.

04 — Normal

🔔 Normal Distribution

The most important continuous distribution — symmetric, bell-shaped, governed by mean μ and std deviation σ. Underpins the Central Limit Theorem.

f(x) = (1/(σ√2π)) · exp[ −(x−μ)² / (2σ²) ] Standard Normal: Z = (X − μ) / σ ~ N(0,1) Empirical Rule: μ ± 1σ → 68.27% μ ± 2σ → 95.45% μ ± 3σ → 99.73%
Example 14

Normal: standardising a score

📘 Definition

Normal(μ,σ²): bell-shaped continuous distribution. Standardise via Z=(X−μ)/σ ~ N(0,1) and use Z-tables.

❓ Example Problem

IQ scores follow N(100, 15²). Find P(IQ > 130) and the IQ that separates the top 5 %.

✅ Solution
P(X>130) = P(Z > (130−100)/15) = P(Z > 2) ≈ 0.0228 (≈ 2.28 %) Top 5 % cut-off: z₀.₉₅ = 1.645 X = μ + z·σ = 100 + 1.645·15 ≈ 124.7
💡 Explanation

A z-score of 2 means 2 SDs above the mean. By the Empirical Rule ~2.5 % lie beyond 2σ, matching 0.0228.

🔍 Interpretation

Only ~2.3 % of people score above 130 (often called "gifted"). To be in the top 5 %, an IQ of ~125 is required.

05 — Bivariate

🔗 Joint, Marginal & Covariance

When two random variables X and Y are studied together, their joint distribution describes how they vary in tandem.

Joint PMF/PDF: f(x,y) Marginal of X: f_X(x) = Σ_y f(x,y) or ∫ f(x,y) dy Conditional: f(y|x) = f(x,y) / f_X(x) Covariance: Cov(X,Y) = E[(X − μ_X)(Y − μ_Y)] = E[XY] − E[X]·E[Y] Correlation: ρ = Cov(X,Y) / (σ_X · σ_Y), −1 ≤ ρ ≤ 1 Independence ⇒ Cov(X,Y) = 0 (converse not always true)
Example 15

Covariance & Correlation

📘 Definition

Covariance measures joint linear variation; Correlation ρ = Cov/(σx·σy) is its scale-free version in [−1,1].

❓ Example Problem

X = hours studied: 2, 4, 6, 8. Y = marks: 50, 60, 70, 90. Find Cov(X,Y) and ρ.

✅ Solution
x̄ = 5, ȳ = 67.5 dx: -3,-1,1,3 dy: -17.5,-7.5,2.5,22.5 dx·dy: 52.5, 7.5, 2.5, 67.5 Σ = 130 Cov(X,Y) = 130/(n-1) = 130/3 ≈ 43.33 Var(X) = (9+1+1+9)/3 = 20/3 → σx ≈ 2.582 Var(Y) = (306.25+56.25+6.25+506.25)/3 ≈ 291.67 → σy ≈ 17.078 ρ = 43.33 / (2.582·17.078) ≈ 0.983
💡 Explanation

We compute deviations from the means, then average their products. Dividing by σx·σy strips the units away.

🔍 Interpretation

ρ ≈ 0.98 → very strong positive linear relationship: more study hours strongly associated with higher marks.

CHAPTER 04 · UNIT II

Statistical Inference — I

From samples to decisions about populations — sampling techniques, confidence intervals, hypothesis testing, and p-values form the toolkit of inferential statistics.

01 — Sampling

🎯 Sampling Methods

A well-chosen sample lets us draw conclusions about an entire population without measuring everyone.

SRSWR

Simple Random Sampling With Replacement. Each unit can be chosen more than once. Trials are independent.

SRSWOR

Simple Random Sampling Without Replacement. Each unit chosen at most once. Lower variance for finite populations.

Stratified

Divide population into homogeneous strata, then sample randomly from each. Improves precision when strata differ.

Cluster

Divide population into clusters, randomly select whole clusters. Cost-effective for geographically dispersed populations.

02 — Hypothesis Testing

⚖️ Basic Terminology

H₀ — Null hypothesis (status quo / no effect) H₁ — Alternative hyp. (the claim to be tested) Errors: Type I (α): reject H₀ when it is true Type II (β): fail to reject H₀ when it is false Power = 1 − β Test statistic — function of sample data Significance level α — typically 0.05 or 0.01 p-value — probability of observed (or more extreme) data under H₀ Decision: reject H₀ if p-value < α
Example 17

Set up H₀, H₁ and identify errors

📘 Definition

H₀ = status quo, H₁ = claim. Type-I (α): reject true H₀; Type-II (β): fail to reject false H₀.

❓ Example Problem

A company claims its bulbs last on average more than 1000 hours. Set up the hypotheses, state α = 0.05, and describe Type-I and Type-II errors in this context.

✅ Solution
H₀: μ ≤ 1000 (no improvement) H₁: μ > 1000 (right-tailed claim) Significance level α = 0.05 Type-I error: Conclude bulbs last >1000 hr when they actually do not. Type-II error: Fail to detect that bulbs really do last >1000 hr.
💡 Explanation

We always start from the conservative H₀. The direction of H₁ (one-tailed > here) determines the rejection region.

🔍 Interpretation

A Type-I error misleads consumers into believing a false marketing claim. A Type-II error denies the company credit for a genuine improvement. α controls the first; power = 1−β controls the second.

03 — Confidence Interval

🎯 Confidence Interval & Sample Size

A confidence interval gives a plausible range for an unknown parameter at a chosen confidence level (usually 95%).

CI for population mean (σ known): x̄ ± z_{α/2} · (σ / √n) CI when σ unknown (use sample s, t-distribution): x̄ ± t_{α/2, n−1} · (s / √n) CI for proportion: p̂ ± z_{α/2} · √[ p̂(1−p̂) / n ] Sample size determination (mean): n = ( z_{α/2} · σ / E )² where E = margin of error Sample size (proportion): n = z²_{α/2} · p(1−p) / E²
Example 18

95 % CI for the mean

📘 Definition

A confidence interval gives a plausible range for an unknown parameter at a chosen confidence level (e.g., 95 %).

❓ Example Problem

A sample of n = 36 light bulbs has x̄ = 1050 hr and population SD σ = 120 hr. Construct a 95 % CI for μ.

✅ Solution
SE = σ/√n = 120/√36 = 20 z_{0.025} = 1.96 Margin of error E = 1.96·20 = 39.2 95 % CI = 1050 ± 39.2 = (1010.8 , 1089.2) hr
💡 Explanation

Because σ is known and n is reasonably large, we use the z-interval. SE shrinks as √n grows — bigger samples, narrower intervals.

🔍 Interpretation

We are 95 % confident the true mean lifetime is between 1010.8 and 1089.2 hours. Since 1000 lies outside this interval, the data give evidence that μ > 1000.

Example 19

One-sample Z test (P-value)

📘 Definition

Use a Z test when σ is known (or n large). Compute Z = (x̄−μ₀)/(σ/√n) and compare its p-value to α.

❓ Example Problem

Test H₀: μ = 1000 vs H₁: μ > 1000 with x̄ = 1050, σ = 120, n = 36, α = 0.05.

✅ Solution
Z = (1050 − 1000) / (120/√36) = 50 / 20 = 2.5 P-value (right-tail) = P(Z > 2.5) ≈ 0.0062 Decision: 0.0062 < 0.05 → REJECT H₀.
💡 Explanation

The test statistic measures how many SEs the sample mean is above μ₀. The p-value is the tail probability under H₀ of seeing something this extreme or more.

🔍 Interpretation

There is strong evidence (p ≈ 0.6 %) that the true mean lifetime exceeds 1000 hours — supporting the company's claim at the 5 % level.
